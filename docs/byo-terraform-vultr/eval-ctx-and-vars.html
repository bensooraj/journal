Evaluation Context and Variables

Evaluation Context​

info

Each argument attribute in a configuration file is interpreted as an expression. In the HCL native syntax, certain basic expression functionality is always available, such as arithmetic and template strings, and the calling application can extend this by making available specific variables and/or functions via an evaluation context. (source: Expression Evaluation)

All expressions or argument attributes up until now have been literal values (as shown in literal.hcl below). Let's now load these values using variables (AMS and env.VULTR_API_KEY) and functions (file(...)) instead (as shown in dynamic.hcl below).

  • literal.hcl
  • dynamic.hcl
griffon {
    region = "ams"
    vultr_api_key = "1234567890"
}
ssh_key "my_key" {
    ssh_key = "ssh-rsa AAAAB3NzaC1yc2EAAAADA"
}
griffon {
    region = AMS // or ${AMS}
    vultr_api_key = env.VULTR_API_KEY
}
ssh_key "my_key" {
    ssh_key = file("~/.ssh/griffon_ssh_key.pub")
}

EvalContext(hcl.EvalContext) is the type used to describe the variables and functions available during expression evaluation, if any. (source: Expression Evaluation | EvalContext)

Variables​

In order for the parser to understand AMS as a variable, we will create an instance of hcl.EvalContext and pass it to hclsimple.Decode(filename string, src []byte, ctx *hcl.EvalContext, target interface{}) error as a the third argument (0863a4b).

parser.go
func ParseHCL(filename string, src []byte, ctx *hcl.EvalContext) (*Config, error) {
    // ...
}

func getEvalContext() *hcl.EvalContext {
    vars := make(map[string]cty.Value)

    // Region variables
    vars["AMS"] = cty.StringVal("ams")

    return &hcl.EvalContext{
        Variables: vars,
    }
}

We can also use these variables as template strings (cfd6327) as shown below,

  • Example 1
  • Example 2
griffon {
    region = "${AMS}terdam" // result: "amsterdam"
    vultr_api_key = "1234567890"
}
griffon {
    region = "${AMS == "ams" ? "toronto" : "amsterdam"}" // result: "toronto"
    vultr_api_key = "1234567890"
}

Nested Variables​

Say we want to load Vultr's API key from the environment,

griffon {
    region = "${AMS == "ams" ? "toronto" : "amsterdam"}" // result: "toronto"
    vultr_api_key = env.VULTR_API_KEY
}

We will use the method func cty.ObjectVal(attrs map[string]cty.Value) cty.Value to further nest key-value pairs and attach to (hcl.EvalContext).Variables as shown below (650a749),

parser.go
func getEvalContext() *hcl.EvalContext {
    // ...
    // Environment variables
    vars["env"] = cty.ObjectVal(map[string]cty.Value{
        "VULTR_API_KEY": cty.StringVal(os.Getenv("VULTR_API_KEY")),
    })

    return &hcl.EvalContext{
        Variables: vars,
    }
}
